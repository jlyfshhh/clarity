What it does

Aquatic care without the scattered notebooks

Clarity is organized around aquariums and ponds, because water, equipment, maintenance, and livestock groups all shape the same system.

✅

Today’s care

Feeding, top-offs, water changes, testing, filter maintenance, dosing, and recurring routines.

🧪

Water readings

Temperature, pH, ammonia, nitrite, nitrate, salinity, alkalinity, calcium, magnesium, and more.

🐠

System records

Volume, equipment, lighting, livestock groups, consumables, changes, and notes with the physical tank.

👑

Owner controls

Owners manage routines, parameter targets, household access, exports, and system structure.

🧤

Zookeeper updates

Family members record feeding, maintenance, and test results from their own phones.

📦

Portable by design

Stable tank IDs, numeric readings, ordinary tables, ISO dates, and open exports avoid lock-in.

Good to know

Questions keepers ask

Is Clarity ready to install?

Not yet. The working prototype establishes the interface and data structure. Editable routines, parameter targets, household authentication, automatic backups, and the Raspberry Pi installer are the next major milestones.

Does Clarity assume clear water is healthy?

No. Visual clarity is only an observation. The app is designed to keep actual test results, trends, maintenance, and system changes together so “looks fine” is not mistaken for measured conditions.

Will every tank use the same parameters?

No. A reef, freshwater community, betta tank, turtle habitat, and pond need different tests and targets. Clarity’s data model is tank-specific rather than prescribing one universal panel.

Can several family members use it?

That is the design. An Owner manages the system while Zookeepers can record care and readings from their own phones. One shared server keeps everyone on the same current state.

Will my records be trapped in the app?

No. Portability is a core requirement: stable identifiers, ordinary relational data, numeric values, ISO timestamps, and open JSON and CSV exports. The Pi edition will use SQLite with dated backups.
